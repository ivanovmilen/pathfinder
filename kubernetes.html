Check the current database version with rladmin status extra all or REST API curl -k -u <user>:<password> https://<cluster-host>:9443/v1/bdbs | jq '.[] | {name, uid, redis_version}' Example returned redis_version: 7.4.3
Active-Active databases, also known as CRDB (Conflict-free Replicated Databases), replicate data across multiple Redis Enterprise clusters in different geographic locations. Each participating cluster holds a full copy of the data and can accept both read and write operations locally. Upgrading an Active-Active deployment requires a dedicated procedure that coordinates the upgrade across all participating clusters. All clusters must be upgraded to the target Redis Enterprise version before the CRDB configuration itself is upgraded.
